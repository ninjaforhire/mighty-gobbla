# Coding Conventions

## Naming Conventions
- **Variables**: snake_case (Python), camelCase (JS/TS)
- **Functions**: snake_case (Python), camelCase (JS/TS)
- **Classes**: PascalCase

## Code Style
- Use 4 spaces for indentation (Python).
- Use 2 spaces for indentation (JS/TS).
- Always include type hints where possible (Python type hinting, TypeScript).
- Add docstrings to all public functions and classes.

## Git Workflow
- Commit messages should be descriptive (e.g., "feat: add user login", "fix: resolve memory leak").
- Create feature branches for new tasks.

## Error Handling
- Use try/catch blocks for external API calls and file I/O.
- Log errors meaningfully.
